# Lever-LM（Qwen2-VL 版）项目总结

**日期：** 2026-06-05  
**代码目录：** `/home/jiyi/lizhiheng/Lever-LM`  
**基线：** 原版 [Lever-LM](https://github.com/ForJadeForest/Lever-LM)（ICLR 2024，InfoScore + 自回归 ICD 选择）

---

## 一、项目目标

在 **不微调 Qwen2-VL-2B-Instruct** 的前提下，训练一个小型 **示例选择器（PointerSelector）**，为每个 query 从候选池检索并 **有序** 选出 in-context 示例（ICD），使大模型在 VQA / Caption / SST-2 等任务上的 **InfoScore（信息增益）** 最大。

用户明确提出的三个核心诉求：

1. 解决 Step1 数据里 **reward = −∞** 泛滥（v1 约 44.5%）
2. 解决 **`[3,9]` 与 `[9,3]` 顺序不可区分** 的架构缺陷
3. 对齐原版 **三任务**（VQA + Caption + SST-2）管线

---

## 二、v1 存在的主要问题（改进前）

| 问题 | 表现 | 根因 |
|------|------|------|
| reward = −∞ | 5000 条 beam 数据里 **44.5%** 全 −∞ | 在 prompt 里用子串搜 GT 答案算 log-prob，长答案/多样答案匹配失败 |
| 顺序不可区分 | `P([3,9]) = P([9,3])` | 旧 PointerSelector 一套 logits 求和，**置换不变** |
| 数据有偏 | anchor = 训练集前 5000 条顺序取 | 顺序偏差 + anchor 自当候选池（互引） |
| 嵌入维度错误 | d_model=768 | Qwen2-VL 实际 hidden 维 **1536** |
| 任务单一 | 仅 VQA | 无 Caption / SST-2 |
| 0-shot 评测假象 | ICL 初评 0-shot acc=0% | 长句生成 + 严格匹配，未做短答处理 |

## 三、v2 全部代码改动（按模块）

### 3.1 数据生成 `generate_data.py`

| 改动 | 说明 |
|------|------|
| **Teacher-forcing 打分** | GT 答案拼到 prompt 末尾，读 token log-prob，不再搜子串 |
| **InfoScore reward** | `--reward_mode info`：`logP(y\|S,x) − logP(y\|x)` |
| **随机 anchor + 解耦池** | `--anchor_random`；VQA 池 2 万条，与 5000 anchor 不重叠 |
| **断点续跑** | `--resume`、`--pool_in`、JSON 原子写入 |
| **多任务** | `--task {vqa,caption,sst2}` |
| **Beam / K** | beam=3（经济档），检索 K=32 |

### 3.2 大模型打分 `lever_lm/qwen_vl_scorer.py`

| 改动 | 说明 |
|------|------|
| **`score_batch_tf()`** | Teacher-forcing 批量 InfoScore，消除 −∞ |
| **`embed()`** | Qwen hidden mean-pool → 1536 维检索向量 |
| **无图模式** | `image=None` 支持 SST-2 纯文本 |

### 3.3 选择器 `lever_lm/pointer_selector.py`（核心架构改造）

**v1：** 单次前向 → 一套 pointer logits → log-prob 求和（顺序无关）

**v2：** **自回归指针解码器**

```
输入序列: [query_emb, sel_1, sel_2, ...]
每步: 因果 self-attn + cross-attn 到 K 个候选
输出: P(icd_t | query, icd_{<t})
→ P([3,9]) = P(3|q) · P(9|q,3) ≠ P([9,3])
```

单元测试 `scripts/test_order_aware.py`：**5/5 PASS**，`mean|ΔlogP| ≈ 0.93`

### 3.4 训练 `train.py` + `lever_lm/lever_lm_module.py`

| 改动 | 说明 |
|------|------|
| **RCE 损失** | 对 beam 序列做 reward-weighted CE |
| **池负样本 InfoNCE** | `--neg_weight 1.0`、`--pool_file`；随机池序列作负例 |
| **效果** | VQA 排列对顺序准确率 **0.609 → 0.739** |

### 3.5 多任务注册 `lever_lm/tasks.py`

统一数据格式 `{id, image?, question, answer}`，注册 VQA / COCO Caption / SST-2 三种任务的数据加载与 prompt 模板。

### 3.6 下游 ICL 评测 `icl_inference.py`

| 改动 | 说明 |
|------|------|
| **完整 VQA ICL 链路** | 嵌入 → top-K 检索 → PointerSelector 选择 → Qwen 生成 → VQA acc |
| **Baseline** | `--baseline random` / `zeroshot` |
| **Shot sweep** | `--sweep`：shot 1～8，输出 Avg:1~8 |
| **统一评测协议** | `--eval_protocol unified`（默认）：三方法均加短答后缀 + 答案抽取 + strict 匹配 |
| **旧协议保留** | `legacy_lever_strict` / `legacy_zeroshot_relaxed` |

### 3.7 评测与脚本

|------|------|

## 七、为什么 Lever 低于 0-shot？（重要分析）

### 7.1 数字事实

- Lever **稳定优于 Random**（5000：+1.7%），说明 selector **并非无效**
- Lever **低于 0-shot**（5000：63.4% vs 70.5%，差 7.1%）

### 7.2 原因（不主要是「只有 5000 anchor」）

训练 anchor 5000 **与原版一致**。主要瓶颈：

1. **候选池太小（2 万 / 44 万 ≈ 4.5%）**  
   answer_overlap@32 仅 ~60%，约 40% query 的 top-32 里没有 GT 同类答案。

2. **错 ICD 伤害 > 对 ICD 收益**（5000 统计）  
   - Lever 错 & 0-shot 对：**549 条**  
   - Lever 对 & 0-shot 错：**191 条**  
   - 净损失约 **358 条**

3. **Selector 在 VQA 上信号弱**  
   select 并集命中 ≈ 0.15（≈ 随机 0.19）；beam reward 打平，只能做边际改进。

4. **Shot 越多越差（负迁移）**  
   Lever：65.2%（1-shot）→ 60.3%（8-shot），多塞噪声示例反而降分。

5. **模型与 setup 差异**  
   Qwen2-VL-2B 对噪声 ICD 更敏感；原版 9B + 全量 FAISS 更容易从 ICL 受益。

## 十一、关键命令速查

```bash
# Step1 数据生成（VQA 示例）
python generate_data.py --task vqa --reward_mode info --anchor_random \
  --num_anchors 5000 --pool_size 20000 --beam_size 3 --K 32 --resume

# Step2 训练
python train.py --run_name lever_lm_v2_poolneg \
  --train_file data/vqav2_train_beams_v2_train.json \
  --pool_file data/vqav2_pool_v2.json --neg_weight 1.0

# Selector 评测
python scripts/eval_v2.py --ckpt checkpoints/lever_lm_v2_poolneg/epoch003-valloss7.0768.ckpt \
  --beam_file data/vqav2_train_beams_v2_val.json --pool_file data/vqav2_pool_v2.json

# Step3 ICL（统一协议）
python icl_inference.py --pool_file data/vqav2_pool_v2.json --K 32 --shot_num 2 \
  --split validation --max_samples 5000 --eval_protocol unified \
  --ckpt_path checkpoints/lever_lm_v2_poolneg/epoch003-valloss7.0768.ckpt \
  --output_file results/icl_v2_lever_unified_n5000.json

# Shot sweep
python icl_inference.py --pool_file data/vqav2_pool_v2.json --K 32 --sweep \
  --split validation --max_samples 5000 --eval_protocol unified \
  --ckpt_path checkpoints/lever_lm_v2_poolneg/epoch003-valloss7.0768.ckpt \
  --output_file results/icl_v2_lever_unified_n5000.sweep.json
```
